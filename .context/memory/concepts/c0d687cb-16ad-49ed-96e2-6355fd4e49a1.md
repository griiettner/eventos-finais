# Environment Configuration

# Environment Configuration

Eventos Finais project environment variables and configuration settings.

## Kinde Authentication Variables
```
VITE_KINDE_DOMAIN=https://your-domain.kinde.com
VITE_KINDE_CLIENT_ID=your-client-id
VITE_KINDE_REDIRECT_URL=http://localhost:5173/dashboard
VITE_KINDE_LOGOUT_URL=http://localhost:5173/login
VITE_KINDE_APPLE_CONNECTION_ID=your-apple-connection-id
VITE_KINDE_FACEBOOK_CONNECTION_ID=your-facebook-connection-id
VITE_KINDE_GOOGLE_CONNECTION_ID=your-google-connection-id
VITE_KINDE_EMAIL_CODE_CONNECTION_ID=your-email-connection-id
```

## Memory Configuration
```json
{
  "memoryDir": ".context/memory",
  "enabled": true,
  "description": "Eventos Finais project memory"
}
```

## Development Environment
- **Framework**: Vite with React 19 and TypeScript
- **Build Tool**: Vite (Rolldown for faster builds)
- **Package Manager**: npm
- **Browser Target**: Modern browsers with ES2020+ support

## PWA Configuration
- **Service Worker**: Enabled via Vite PWA plugin
- **Offline Support**: Full offline functionality
- **Install Prompt**: Custom install experience
- **Cache Strategy**: Network-first for critical assets

## Security Considerations
- All Kinde credentials stored in environment variables
- No hardcoded secrets in source code
- Role-based access control via Kinde
- HTTPS required for production deployment

## Database Configuration
- **SQLite WASM**: In-browser database
- **Storage**: OPFS (Origin Private File System) when available
- **Fallback**: In-memory storage for older browsers
- **Worker**: Database operations in separate thread

## Development Notes
- Use `.env.local` for local development overrides
- Environment variables must be prefixed with `VITE_` for client-side access
- Kinde dashboard configuration required for OAuth connections
- Memory system configured to use `.context/memory` directory
